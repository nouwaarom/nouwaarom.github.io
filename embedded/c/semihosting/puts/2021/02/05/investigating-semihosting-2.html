This is part two of a series, part one can be found here. In this post we will investigate how we can print using semihosting. In particular, we will explore how puts works.

Printing

When we look at the disassembly of main, which can be found by $ arm-none-eabi-objdump --disassemble main.o. We see that the printf has been replaced by a call to puts.

Let us look at the source of puts.

Let us first look at the preprocessor statements. There is a conditional compilation on _FVWRITE_IN_STREAMIO. With some googeling we find that this flag can be used to disable the io vector buffer. source It turns out the buffer is available for newlib but not for newlib nano.
